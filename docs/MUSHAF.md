# وحدة المصحف الشخصي

مصحف داخل المنصّة للمراجعة الذاتية للطالب بعد الحصة. **لا AI ولا تحليل صوت** — المعلم هو من يسجّل آخر موضع والأخطاء يدوياً.

## المكوّنات
- **الطالب** [`/student/mushaf`](../src/app/student/mushaf/page.tsx): عرض بالصفحات (٦٠٤) بطابع مصحفي، تنقّل (سورة/جزء/صفحة/علامات/آخر موضع)، تشغيل صوتي من أي كلمة مع تمييز الكلمة الجارية، ودرج تفاصيل الأخطاء.
- **المعلم** [`/teacher/students/[id]/mushaf`](../src/app/teacher/students/%5Bid%5D/mushaf/page.tsx): عارض تفاعلي — الضغط على كلمة/آية يحدّد الموضع ثم «تعيين آخر موضع» أو «إضافة خطأ هنا»، مع قائمة تعديل/حل/حذف.
- **الأدمن** [`/admin/reciters`](../src/app/admin/reciters/page.tsx): إضافة/تفعيل/حذف القرّاء.

## مصادر البيانات
- **النص**: مصحف عثماني من [alquran.cloud](https://alquran.cloud) — يُجلب **مرة واحدة** ويُضمَّن في `db/seed/quran.json`.
- **الصوت**: تلاوات [quran.com](https://quran.com) مع **توقيتات الكلمات (segments)** للتزامن الدقيق والبدء من أي كلمة. «مصدر» القارئ = معرّف التلاوة الرقمي (الحصري=٦، المنشاوي=٩، عبد الباسط=٢، العفاسي=٧). يُجلب عبر وكيل نفس النطاق [`/api/quran/audio/[recitation]/[page]`](../src/app/api/quran/audio/%5Brecitation%5D/%5Bpage%5D/route.ts).

## التنصيب على قاعدة نظيفة
```bash
# 1) الجداول تُنشأ تلقائياً من db/init/01_schema.sql
#    (quran_surahs, quran_ayahs, reciters, student_mushaf_progress,
#     student_mushaf_mistakes, student_mushaf_bookmarks)

# 2) بذر نص القرآن + القرّاء (يقرأ db/seed/quran.json المُضمَّن)
npm run seed:quran

# (اختياري) إعادة جلب النص من المصدر وتحديث db/seed/quran.json:
npm run fetch:quran
```

> `seed:quran` عديم الأثر عند التكرار (يتخطّى إن وُجدت آيات)، ويفصل البسملة عن الآية الأولى لكل السور عدا الفاتحة، ويزيل BOM.

## ملاحظات مهمة
- **البسملة** عنوان قبل السورة (ليست آية) في كل السور **عدا الفاتحة** حيث هي الآية ١. التوبة بلا بسملة.
- **القرّاء المبذورون**: الحصري المعلّم، المنشاوي، عبد الباسط، العفاسي. (المصدر = مجلد everyayah مثل `Husary_Muallim_128kbps`.)
- **تمييز الكلمة** أثناء التلاوة تقديري (بطول الكلمات) لا توقيت حرفي — «دليل حركة».
- صلاحية إدارة القرّاء: `settings.manage`. إجراءات المعلم مقيَّدة بطلابه. كلها تُسجَّل في الـ audit.
- عارض الطالب يجلب الصفحات بـ `no-store` + كاش في الذاكرة (يتجنّب نسخاً قديمة بعد أي تصحيح للبيانات).

## تخطيط الأسطر (مصحف المدينة)
العرض يطابق **انكسار أسطر مصحف المدينة** (١٥ سطراً/صفحة) عبر جدول `quran_words` المُبذور من `db/seed/quran-layout.json` (بيانات quran.com — رقم السطر لكل كلمة). كل كلمة عنصر مستقل قابل للضغط، فتبقى العلامات والصوت والتمييز كلمة‑بكلمة تعمل.

```bash
npm run seed:quran-layout     # بذر تخطيط الأسطر (من db/seed/quran-layout.json)
npm run fetch:quran-layout    # (اختياري) إعادة جلب التخطيط من quran.com
```

### خط المصحف الدقيق (QCF v2) — اختياري
زر **«خط المصحف»** في عارض الطالب يبدّل إلى التطابق الحرفي لأشكال حروف مصحف المدينة (خطوط QCF v2 لكل صفحة). البيانات: `code_v2` + `v2_page` لكل كلمة في `quran_words`. الخطوط تُجلب من quran.com عبر **وكيل نفس النطاق** [`/api/quran/font/[vpage]`](../src/app/api/quran/font/%5Bvpage%5D/route.ts) (لتفادي حجب CORS على الخطوط). الوضع افتراضياً مُطفأ، ويحافظ على الضغط على الكلمات والعلامات والصوت.

```bash
npm run fetch:quran-layout    # يجلب أيضاً code_v2 + v2_page
npm run seed:quran-layout
```
